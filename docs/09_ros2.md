### Design choices

- **Mode A first** â€” reuses Phase 06 Docker/GHCR server; ROS never loads ONNX (thin client).
- **Same postprocess as API** â€” client decodes JSON masks like `visualize_predict.py`; no duplicate ONNX in ROS for Mode A.
- **`.venv` for pip deps** â€” same convention as Phase 07; `colcon` built with venv Python.
- **`numpy<2` in venv** â€” ROS Jazzy `cv_bridge` is built against NumPy 1.x; pin in `requirements-ros2-client.txt`.
- **`PYTHONNOUSERSITE=1` when running nodes** â€” avoids `~/.local` NumPy 2.x breaking `cv_bridge`.
- **Default score threshold `0.5`** â€” matches training/serving; optional stricter filter via `conf_threshold` param or query string (e.g. `0.65` for fewer boxes).
- **`skip_if_busy: true`** â€” drops camera frames while HTTP inference is in flight (~2 s CPU per frame in tests).
- **JPEG over HTTP** â€” ROS node encodes `bgr8` â†’ JPEG before `POST /predict` (slightly fewer instances vs raw file upload; expected).

### Why Docker is still used (Mode A)

| Component | Runs where | Role |
|-----------|------------|------|
| FastAPI + ONNX | **Docker** `:8000` | Model load, inference, optional Mongo log |
| ROS2 nodes | **Host** (Jazzy) | Topics, bridge, HTTP client |
| `predict_image.sh` / `visualize_predict.py` | **Host** `.venv` | Quick test: send file â†’ save overlay |

Docker is **not** for ROS itself â€” only for the **inference server** built in Phase 06/08. Alternative: run `uvicorn` in `.venv` without Docker (see `docs/06_serving.md`).

## 6. Parameters (`config/params.yaml`)

| Parameter | Default | Notes |
|-----------|---------|-------|
| `server_url` | `http://localhost:8000` | FastAPI base URL |
| `predict_path` | `/predict` | Endpoint path |
| `input_topic` | `/camera/image_raw` | Subscription |
| `output_topic` | `/perception/instance_masks` | Mask publisher |
| `publish_viz` | `true` | Overlay topic |
| `request_timeout_sec` | `120` | Slow CPU inference OK |
| `jpeg_quality` | `90` | ROS â†’ API JPEG encode |
| `skip_if_busy` | `true` | Drop frames during HTTP call |
| `conf_threshold` | `0.5` | Passed as `?conf_threshold=` to API |

API env default: `CONF_THRESH=0.5` (`docker-compose.yml`). Per-request override without restart: `POST /predict?conf_threshold=0.65`.

## Build log â€” issues and fixes

| Issue | Fix |
|-------|-----|
| `cv_bridge` + NumPy 2.x crash | Pin `numpy<2` in `requirements-ros2-client.txt`; rebuild `.venv` |
| `~/.local` NumPy shadows venv | `export PYTHONNOUSERSITE=1` in run scripts / docs |
| `rclpy` logger: `info("a %s", x)` TypeError | Use f-strings only (`info(f"... {x}")`) |
| `colcon` without sourced install | `ros2 pkg not found` â€” always `source install/setup.bash` |
| Too many boxes at threshold 0.5 | Kept default **0.5**; optional `conf_threshold` on API + ROS + `predict_image.sh` |
| JPEG vs raw file instance count | Documented: ROS path ~25 inst., direct file ~31 on same val frame (recompression) |
